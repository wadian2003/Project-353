## Git Commit Rules

- Commit changes at least once per week.
- Use meaningful commit messages.
- Do not upload the entire project in a single commit.
- Every team member must maintain their own GitHub repository.

### Example Commit Messages

```
Initial project structure
Add requirements analysis
Complete use case identification
Create use case diagram
Develop context diagram
Add Level-0 DFD
Complete Level-1 DFDs
Design activity diagrams
Create UML class diagram
Design relational database
Develop low-fidelity prototype
Add Gantt chart
Update mid progress report
Prepare final report
Final review before submission
```
